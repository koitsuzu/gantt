# 甘特圖渲染規範 (Gantt Capability)

## 需求：自定義渲染引擎 (Gantt Renderer)

系統必須支援高效的數據渲染，並支援大規模式時間軸。

### 場景：自定義渲染器初始化
- **當** `GanttRenderer` 被例項化時
- **那麼** 系統應建立 `gantt-chart` 容器及其子容器 `gantt-time-scale` 與 `gantt-rows`

### 場景：單一時間尺度切換
- **當** 用戶從按鈕切換時、日、週、月、季、年之一時
- **那麼** 系統應重新計算橫座標像素與時間的對映
- **且** 更新表頭及所有圖條 (Bars) 的位置

## 需求：圖表互動 (Chart Interaction)

系統必須支援圖條的拖拉操作，反映時間軸的變更。

### 場景：拖動圖條移動
- **當** 用戶在編輯模式下拖拽一個任務圖條時
- **那麼** 系統應將偏移量 (offset) 轉換回日期
- **且** 同步至後端 API `/api/sub-tasks/:id`

### 場景：調整圖條寬度
- **當** 用戶調整圖條右側控制點更改寬度時
- **那麼** 系統應重新計算結束日期 `end_date`
- **且** 確認其符合階段的時間範圍限制

## 需求：視圖縮放與展開 (View Zoom & Expansion)

系統必須能展開或摺疊不同層級的數據。

### 場景：展開至特定層級
- **當** 用戶點擊「1 階 / 2 階 / 3 階 / 全部」按鈕時
- **那麼** 系統應摺疊或展開所有深度對應的任務
- **且** 重新計算圖表區域的動態高度
