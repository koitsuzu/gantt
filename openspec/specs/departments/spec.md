# 部門管理與色標規範 (Departments Capability)

## 需求：部門色標 (Department Color Labels)

系統必須支援部門的自訂顏色，並將其應用於甘特圖中的所有對應任務。

### 場景：新增部門及其顏色
- **當** 管理員在設定中新增部門並選擇一個 hex 顏色 (如 `#ef4444`)
- **那麼** 系統應將此資訊存儲於 `departments` 表

### 場景：任務自動著色
- **當** 用戶在甘特圖中建立子任務且選擇該部門時
- **那麼** `renderer` 在顯示時應讀取部門的 `color`
- **且** 將此顏色應用於任務的橫條背景，而非預設的灰色

## 需求：部門任務模板 (Department Task Templates)

每位部門可關聯特定的常用任務清單。

### 場景：載入部門常用任務
- **當** 在新增子任務彈窗中選擇某一部門時
- **那麼** 系統應獲取對應的 `department_tasks`
- **且** 供用戶在任務主體中快速選擇。

## 需求：管理模式權限 (Admin Mode Only)

部門的新增、修改與刪除僅限於管理員。

### 場景：非管理員嘗試編輯部門
- **當** 非管理員用戶嘗試呼叫 `/api/departments` 介面
- **那麼** 系統應拒絕操作
- **且** 在前端按鈕上隱藏相關功能
