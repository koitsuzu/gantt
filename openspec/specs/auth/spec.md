# 權限驗證規範 (Auth Capability)

## 需求：密碼驗證 (Password Verification)

系統必須支援基於密碼的權限控制，以區分「唯讀」、「編輯」和「管理員」角色。

### 場景：輸入編輯密碼
- **當** 用戶在切換開關時輸入 `EDIT_PASSWORD`（預設為 1234）
- **那麼** 系統應解鎖基礎編輯功能（新增/修改專案與任務）
- **且** 將狀態標記為 `is-editing`

### 場景：輸入管理員密碼
- **當** 用戶輸入 `ADMIN_PASSWORD`（預設為 admin1234）
- **那麼** 系統應解鎖所有功能，包括敏感設置（如部門管理）
- **且** 將角色標記為 `is-admin`

### 場景：密碼錯誤
- **當** 用戶輸入錯誤的密碼
- **那麼** 系統應提示「密碼錯誤」並保持唯讀狀態

## 需求：模式切換 (Mode Switching)

前端介面必須根據當前權限狀態動態調整可見性和可交互性。

### 場景：唯讀模式下
- **當** 系統處於唯讀狀態（預設）
- **那麼** 所有新增、修改和刪除按鈕應被隱藏或禁用
- **且** 甘特圖的拖拉移動功能應被禁用

### 場景：編輯模式下
- **當** 系統處於編輯/管理員模式
- **那麼** 相應的操作按鈕應顯示
- **且** 允許對合法的專案與任務進行互動調整
