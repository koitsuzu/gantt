# 數據導入導出規範 (Data Portability Capability)

## 需求：CSV 批量導入 (CSV Batch Import)

系統必須支援透過 CSV 檔案一次性建立整個專案結構。

### 場景：上傳合法 CSV
- **當** 用戶上傳一個包含專案名稱、階段、任務、日期及部門的 CSV
- **那麼** 系統應逐行解析數據
- **且** 按順序建立 `projects`、`stages` 與 `sub_tasks`
- **且** 自動處理父子任務的關聯性。

### 場景：CSV 解析錯誤
- **當** CSV 格式不符或日期格式不正確時
- **那麼** 系統應提示「導入失敗」
- **且** 提供具體的錯誤行號與原因。

## 需求：Excel 導出 (Excel Export)

系統必須支援將當前甘特圖狀態導出為含樣式的 Excel 文件。

### 場景：導出單個專案 Excel
- **當** 用戶點擊「匯出 Excel (.xlsx)」
- **那麼** 系統應產生包含專案名稱、概覽、分攤進度及各階段細節的試算表
- **且** 使用顏色標識部門分類。

### 場景：導出成功
- **當** Excel 檔案生成完成
- **那麼** 系統應觸發瀏覽器的下載行為
- **且** 格式應具備良好的表頭鎖定及易讀性
